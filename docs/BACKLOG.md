# punch 백로그 — v1에서 의도적으로 뺀 것들

이 문서는 설계 문서(`docs/superpowers/specs/`)와 의도적으로 분리되어 있다.
"지금 할 것"과 "나중에 할 것"을 섞지 않기 위해서다.
각 항목은 **왜 뺐는지**와 **다시 꺼낼 조건**을 함께 기록한다.

## 1. 일일보고 자동 정리 (Gmail/캘린더 연동 + AI 요약)

- **내용**: Google OAuth 로그인 → 캘린더 일정·Gmail을 자동 수집 + 수동 메모 →
  AI가 일일보고로 정리. 연동은 커넥터(플러그인) 규격으로 만들어
  GitHub/Notion/Slack/Jira 등을 나중에 끼울 수 있게 함
- **뺀 이유**: 브레인스토밍 결과 진짜 필요는 출퇴근 기록뿐이었다. 이 기능은
  "출퇴근만 하기엔 민망해서" 붙인 것. 서버 + OAuth + AI 비용이 따라와 프로젝트가
  몇 시간짜리에서 몇 주짜리로 커지고, 완성 못 할 위험이 컸다
- **참고했던 제약**: Gmail 본문 읽기는 Google 민감 권한이라 공개 앱으로 100명
  초과 시 유료 보안 심사 필요
- **다시 꺼낼 조건**: punch를 한 달 이상 매일 실사용하고, 일일보고 작성이 실제로
  반복되는 고통임이 확인될 때

## 2. 앱 내 리마인더 (PWA + 브라우저 푸시 알림)

- **내용**: "매일 9시/18시에 찍으라고 알림" 을 앱이 직접 보냄
- **뺀 이유**: 웹 표준만으로 예약 알림 불가 → 푸시 서버가 필요해서 서버리스 구조가
  깨짐. 게다가 iOS는 홈화면 PWA에서만 알림 가능하고 그마저 자주 끊겨 신뢰도가 낮다.
  "깜빡함 방지"가 목적인데 알림이 가끔 안 오면 목적 실패. 폰 기본 알람이 비용 0에
  신뢰도 ~100%로 우월
- **다시 꺼낼 조건**: 실사용 중 "폰 알람으로는 부족하다"가 실제로 반복해서 느껴질 때

## 3. 탭 하트비트 근무시간 자동 감지

- **내용**: 앱 탭을 열어두면 1분마다 활동 신호를 기록해 실제 근무시간을 자동 추정
- **뺀 이유**: 탭을 닫으면 멈추는 근사치라 정확하지 않고, v1 가치가 불명확.
  화면 켜짐을 진짜로 감지하려면 데스크톱 상주 앱이 필요한데, 이는 "설치 없이
  URL로 누구나"라는 목표와 충돌
- **다시 꺼낼 조건**: 버튼 기록만으로 부족하다는 실사용 피드백이 있을 때

## 4. 기기 간 동기화

- **내용**: 폰과 노트북의 기록을 하나로 합침
- **뺀 이유**: 서버(또는 계정 시스템)가 필요해 서버리스 구조가 깨짐. v1은 JSON
  백업/복원으로 수동 이동 가능
- **다시 꺼낼 조건**: 실사용 중 두 기기를 오가며 찍는 패턴이 실제로 생겨서
  백업/복원이 번거로워질 때

## 5. 데스크톱 상주 앱 (화면 켜짐 감지)

- **내용**: 메뉴바 상주 프로그램이 OS 수준에서 화면 켜짐/꺼짐을 감지해 자동 기록
- **뺀 이유**: 각자 설치가 필요해 "URL만으로 누구나"라는 배포 목표와 정면 충돌
- **다시 꺼낼 조건**: 개인 전용 도구로 방향이 바뀔 때
